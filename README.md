# Название проекта

Видеохостинг платформа.

# Краткое описание идеи проекта

Разработать проект, реализующий аналог известных видеохостинг платформ (например, rutube, vk video). Приложение должно предоставлять возможность пользователю смотреть, оценивать и комментировать видео, а также создать канал, на который можно выкладывать свои видео. Будет присутстовать простая система уведомлений.

# Краткое описание предметной области с перечислением основных сущностей предметной области

В приложении **пользователи** создают **каналы**, выкладывают, просматривают и оценивают **видео**, оставляют **комментарии**, получают **уведомления**.

Основные сущности:
- **Роль** (`roles`) — определяет права доступа пользователя в системе (гость, пользователь, модератор, администратор).
- **Пользователь** (`users`) — зарегистрированный участник платформы с учётной записью.
- **Канал** (`channels`) — именованная страница пользователя, на которую загружаются видео. Пользователь может создать несколько каналов.
- **Видео** (`videos`) — загруженный видеоролик, принадлежащий каналу.
- **Плейлист** (`playlists`) — именованная коллекция видео, создаваемая каналом.
- **Комментарий** (`comments`) — текстовое сообщение пользователя под видео; поддерживается вложенность (ответы на комментарии).
- **Уведомление** (`notifications`) — системное сообщение для пользователя (о новом видео, ответе на комментарий и т.д.).
- **Просмотр** (`watch_history`) — запись факта просмотра видео пользователем.
- **Подписка** (`users_channels_subscriptions`) — связь пользователя с каналом для получения обновлений.
- **Оценка видео** (`users_videos_ratings`) — лайк/дизлайк пользователя к видео.
- **Оценка комментария** (`users_comments_ratings`) — лайк/дизлайк пользователя к комментарию.

# Краткое обоснование целесообразности и актуальности проекта

Платформа предлагает возможность создания нескольких каналов одним пользователем, что позволяет разделять контент по тематикам. Поддержка оценок комментариев (лайк/дизлайк) расширяет взаимодействие с контентом.

# Краткое описание акторов (ролей)

| Актор             | Описание                                                                                                              |
|-------------------|-----------------------------------------------------------------------------------------------------------------------|
| **Гость**         | Неавторизованный посетитель платформы. Может просматривать видео, читать комментарии, выполнять поиск и регистрироваться. |
| **Пользователь**  | Зарегистрированный и авторизованный участник. Может создавать каналы, загружать видео, оценивать контент, комментировать, подписываться на каналы, управлять плейлистами и уведомлениями. |
| **Модератор**     | Пользователь с расширенными правами. Может просматривать жалобы на контент, удалять видео и комментарии, нарушающие правила платформы. |
| **Администратор** | Пользователь с полными правами в системе. Управляет учётными записями пользователей, назначает роли, создаёт и удаляет роли, блокирует нарушителей. |

# Пользовательские сценарии

### Сценарий 1 - Гость: Просмотр видео и регистрация на платформе
1. Гость открывает главную страницу платформы и видит список видео.
2. Гость вводит поисковый запрос в строку поиска.
3. Система отображает список видео, соответствующих запросу.
4. Гость выбирает видео из списка и переходит на страницу просмотра.
5. Система отображает видеоплеер, описание видео, название канала, счётчики и комментарии.
6. Гость просматривает видео и читает комментарии.
7. Гость хочет оставить комментарий или поставить оценку, но система информирует, что для этого необходима авторизация.
8. Гость нажимает кнопку «Регистрация».
9. Система отображает форму регистрации с полями: имя пользователя, email, пароль.
10. Гость заполняет форму и отправляет данные.
11. Система авторизует пользователя и перенаправляет обратно на страницу видео.

### Сценарий 2 - Пользователь: Создание канала, загрузка видео и управление плейлистами
1. Пользователь переходит в раздел «Мои каналы».
2. Пользователь нажимает «Создать канал».
3. Пользователь заполняет название канала, описание и подтверждает создание.
4. Пользователь переходит на страницу своего канала и нажимает «Загрузить видео».
5. Пользователь заполняет название, описание, видеофайл, превью.
6. Всем подписчикам канала приходит уведомление о новом видео.
7. Пользователь переходит в раздел «Плейлисты» канала и нажимает «Создать плейлист».
8. Пользователь заполняет название и описание плейлиста.
9. Пользователь добавляет видео в плейлист.
10. Пользователь может редактировать содержимое плейлиста или удалить его.

### Сценарий 3 - Пользователь: Просмотр видео, подписка, оценка и комментирование
1. Пользователь находит видео и открывает его страницу.
2. Система записывает факт просмотра в историю.
3. Пользователь подписывается на канал автора.
4. Пользователь ставит оценку видео (лайк/дизлайк).
5. Пользователь пишет комментарий под видео.

### Сценарий 4 - Модератор: Модерация контента и комментариев
1. Модератор просматривает видео/комментарий.
2. Модератор удаляет контент, нарушающий правила платформы.

### Сценарий 5 - Администратор: Управление пользователями и ролями
1. Администратор открывает панель администрирования.
2. Администратор просматривает список пользователей и их профили.
3. Администратор назначает роли пользователям (например, повышает до модератора).
4. Администратор удаляет пользователя.
